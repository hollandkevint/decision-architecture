# Diagnose Problem

Run the `problem-framing` skill through all 9 steps. Emphasis on diagnosis before solutions.

## Steps

1. Ask: "What problem are you trying to solve? Describe it in one sentence."
2. Walk through all 9 steps of the `problem-framing` skill.
3. Do NOT skip to solutions before Steps 1-4 (Define, Is/Is Not, Root Causes, Force Field) are complete.
4. Generate solutions only after diagnosis is thorough.
5. Produce the full problem diagnosis document.
6. Save as `problem-diagnosis-<problem-slug>.md` in the current directory.

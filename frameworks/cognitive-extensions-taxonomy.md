# Cognitive Extensions Taxonomy

Tools for thinking mapped to cognitive modes. Each mode requires a different kind of support.

## The Map

| Cognitive Mode | Tools | Decision Session Type |
|----------------|-------|-----------------------|
| **Thinking about Value** | Jobs to Be Done, Value Proposition Canvas | Validate Idea — "Is this worth building?" |
| **Thinking with People** | Co-Design, Research Synthesis | Find Market — "Who needs this and how do I reach them?" |
| **Thinking in Time** | Journey Map, Future State Vision | Go-to-Market — "How does this unfold over time?" |
| **Thinking through Constraints** | Assumption Testing, Feasibility Matrix | Kill Decision — "Should I stop?" |

## Why This Matters

Different decisions require different cognitive modes. A kill decision requires constraint-based thinking, not ideation. A market-finding session requires people-centered thinking, not financial modeling.

Matching the cognitive mode to the decision type prevents the most common failure: applying the wrong kind of thinking to the problem.

## Application

When starting a decision session, identify the cognitive mode first:
1. Am I evaluating value? → Validation tools
2. Am I understanding people? → Research/synthesis tools
3. Am I planning over time? → Journey/sequencing tools
4. Am I testing against constraints? → Assumption/feasibility tools

Then select the skill that matches.
